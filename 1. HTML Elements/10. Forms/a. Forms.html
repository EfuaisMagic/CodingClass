Forms

The form element has no visual impact in and of iteself.
It represents a section of fields
for submitting user information

Form Attributes

  • <form attribute="...">
  • <form>
</form>
  • action = specifies where (file on server) to send form-data when form is submitted.
    • If the action attribute is omitted, the action is set to the current page.
  • target = specifies where to display response that is received after submitting
    • _blank the response is displayed in a new window or tab
    • _self the response is displayed in the current window
    • _parent the response is displayed in the parent frame
    • _top the response is displayed in the full body of the window
    • framename the response is displayed in a named iframe
      • The default value is _SELF which means that the response will open in the current window.
  • name = specifies name of the form
  • method = specifies the HTTP method to use when sending form-data
    • get form data can be sent as URL variables
    • post form data can be sentas HTTP post transaction
      • The default value is GET.
  • autocomplete = specifies whether form should have autocomplete on or offe
    • on
    • off
  • enctype- specifies how form-data should be encoded when submitted to server (only for method="post")
  • accept-charset -Specifies character encodings used for form submissiont
  • novalidate- specifies form should not be validated when submitted
  • rel- specifies relationship between linked resource and current document

  • fieldset-Groups related elements in a form

  • optgroup-Defines a group of related options in a drop-down list
  • output-Defines the result of a calculation
  • option-Defines an option in a drop-down list
  • legend-Defines a caption for a fieldset element
  • datalist-Specifies a list of pre-defined options for input controls
  • select-Defines a drop-down list
  • textarea-Defines a multiline input control (text area)
  • input-One of the most used form elements. Defines an input control
